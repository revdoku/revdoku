Getting started

Build with Atlas

The Atlas API gives your team a predictable way to create projects, capture events, and keep operational data in sync.

Every API response includes a request ID. Keep it when diagnosing an integration.

Authentication

Send your access token as a Bearer token in the Authorization header. Keep tokens on the server and rotate them regularly.

Make your first request

List the projects visible to the current account. Results are paginated and ordered by creation time.

curl https://api.example.test/v1/projects \
  -H "Authorization: Bearer $TOKEN"

{
  "data": [{
    "id": "prj_8fb2",
    "name": "Northstar"
  }]
}
